# Card Pay — BFF Spec

## Overview

Receives the client's credit card payment request, validates the idempotency key, then
forwards the request to Mock NCCC Service. On a successful Phase 1 authorization,
returns a `challengeRef` and `pointsPreview` so the client can display the OTP input
screen with the points-offset summary. This is the first step of the two-phase card
payment Saga.

## API Definition — BFF

### Endpoint

| Field | Value |
|-------|-------|
| Method | POST |
| Client URL | `/api/v1/card/pay` |
| BFF Received Path | `/card/pay` |
| Auth Required | Yes (JWT Bearer Token) |

### Request Header

| Header | Required | Description |
|--------|----------|-------------|
| Authorization | ✅ | `Bearer {accessToken}` |
| Idempotency-Key | ✅ | UUID v4, generated by client |

### Request Body

| Field | Type | Required | Validation | Description |
|-------|------|----------|------------|-------------|
| cardId | String | ✅ | @NotBlank | Member's virtual card identifier (Snowflake ID as String) |
| amount | BigDecimal | ✅ | @NotNull, @DecimalMin("0.01") | Transaction amount |
| currency | String | ✅ | @NotBlank, @Size(min=3,max=3) | Currency code, e.g. `TWD`, `USD` |
| merchantId | String | ✅ | @NotBlank | Merchant identifier |

### Response Body

| Field | Type | Description |
|-------|------|-------------|
| status | String | `CHALLENGE_REQUIRED` / `DECLINED` |
| challengeRef | String | OTP session reference; present when `status = CHALLENGE_REQUIRED` |
| pointsPreview.pointsToUse | Integer | Server-computed points offset (informational only; reserve already set) |
| pointsPreview.cardAmount | BigDecimal | Amount charged to card after points offset |
| declineReason | String | Decline reason code; present when `status = DECLINED` |

## BFF UseCase Flow

**UseCase:** `CardPayUseCase.pay(CardPayRq)`

| Step | Type | Description | Failure Handling |
|------|------|-------------|------------------|
| 1 | `[VALIDATE]` | Validate JWT; extract and set `X-User-Id` header downstream | Invalid / missing token → 401 UNAUTHORIZED; publish L1 log |
| 2 | `[REDIS WRITE]` | SETNX `idempotency:{Idempotency-Key}` with `PROCESSING` status (TTL 60s) | Key exists & `COMPLETED` → return cached response; Key exists & `PROCESSING` → 409 CONFLICT; publish L1 log |
| 3 | `[VALIDATE]` | Validate request body fields | Validation failure → INVALID_REQUEST; delete idempotency key; publish L1 log |
| 4 | `[FEIGN]` | Call Mock NCCC Service → [spec/mock-nccc/pay.md](../../mock-nccc/pay.md) | Downstream error → map to BFF error code; delete idempotency key; publish L1 log |
| 5 | `[KAFKA]` | Publish `operation-log.card.pay` event (async — failure does not affect main flow) | — |
| 6 | `[REDIS WRITE]` | Update `idempotency:{Idempotency-Key}` to `COMPLETED`, cache response (TTL 24h) | — |
| 7 | `[RETURN]` | Return `{ status, challengeRef?, pointsPreview?, declineReason? }` | — |

## Error Codes

| HTTP Status | Error Code | Description | Trigger Condition |
|-------------|------------|-------------|-------------------|
| 400 | INVALID_REQUEST | Malformed or missing request fields | Request body validation failure |
| 401 | UNAUTHORIZED | Authentication failed | Invalid or missing JWT |
| 409 | CONFLICT | Duplicate in-flight request | `Idempotency-Key` is in `PROCESSING` state |
| 422 | CARD_NOT_FOUND | Card does not exist | Card Service CA00001 |
| 422 | CARD_NOT_ACTIVE | Card is not in ACTIVE status | Card Service CA00002 |
| 422 | CARD_EXPIRED | Card has expired | Card Service CA00003 |
| 422 | CARD_FROZEN | Card is currently frozen | Card Service CA00004 |
| 422 | CARD_CANCELLED | Card has been cancelled | Card Service CA00005 |
| 422 | SINGLE_TXN_LIMIT_EXCEEDED | Amount exceeds single-transaction limit | Card Service CA00006 |
| 422 | DAILY_LIMIT_EXCEEDED | Amount exceeds daily cumulative limit | Card Service CA00007 |
| 422 | INSUFFICIENT_FUNDS | Wallet balance is insufficient after points offset | Wallet Service error |
| 500 | INTERNAL_ERROR | Unexpected system error | Unhandled exception |
