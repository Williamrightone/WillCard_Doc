# Card Pay — BFF Spec

> **Service:** bff
> **Related biz spec:** [txn-orch/card-pay.md](../txn-orch/card-pay.md)
> **Last Updated:** 2026-03

---

## 1. Overview

Receives the payment initiation request from the App.
Validates the request, applies idempotency control, publishes an L1 operation log, then forwards the request to Mock NCCC via Feign.
Returns `challengeRef` and estimated amount preview to the App; the App must proceed to `POST /api/v1/card/pay/confirm` to complete the payment.

> This endpoint is **Leg 1** of a two-leg transaction flow. The `challengeRef` returned here binds the OTP session and must be passed to the confirm endpoint within **180 seconds**.

| Scenario | result | failReason / errorCode |

## 4. API Definition — BFF

### Endpoint

| Field | Value |
|-------|-------|
| Method | POST |
| Client URL | `/api/v1/card/pay` |
| BFF Received Path | `/card/pay` |
| Auth Required | Yes (Bearer JWT) |

### Request Header

| Header | Required | Description |
|--------|----------|-------------|
| Authorization | ✅ | Bearer {accessToken} |
| Idempotency-Key | ✅ | UUID v4 generated by client; prevents duplicate submission within 60s |

### Request Body (`bff.contract.dto.CardPayBffRq`)

| Field | Type | Required | Validation | Description |
|-------|------|----------|------------|-------------|
| cardId | String | ✅ | @NotBlank | Card to charge (Snowflake ID as String) |
| amount | Long | ✅ | @NotNull; @Min(1) | Transaction amount in cents |
| currency | String | ✅ | @NotBlank; @Size(min=3,max=3) | ISO 4217 currency code |
| merchantId | String | ✅ | @NotBlank | Merchant identifier |
| usePoints | Boolean | ✅ | @NotNull | Whether to apply points offset |

### Response Body (`bff.contract.dto.CardPayBffRs`)

| Field | Type | Description |
|-------|------|-------------|
| challengeRef | String | UUID; pass to `/card/pay/confirm`; valid for 180s |
| pointsToUse | Long | Points to be deducted in cents (`0` if usePoints = false or no available balance) |
| estimatedTwdAmount | Long | TWD equivalent of txn amount (after FX conversion if overseas; before points deduction) |

## 5. BFF UseCase Flow

**UseCase:** `CardPayUseCase.initiate(CardPayBffRq)` (bff)

| Step | Type | Description |
|------|------|-------------|
| 1 | `[VALIDATE]` | Validate request (@Valid); on failure → throw 400 INVALID_REQUEST |
| 2 | `[HEADER]` | Extract `X-Member-Id` from Security Context |
| 3 | `[REDIS READ]` | GET `bff:pay:idem:{userId}:{idempotencyKey}`; key exists → throw 409 DUPLICATE_REQUEST |
| 4 | `[REDIS WRITE]` | SET `bff:pay:idem:{userId}:{idempotencyKey}` = `1` EX 60 |
| 5 | `[KAFKA]` | Publish `operation-log.card.pay` (L1) — result determined after Step 6 |
| 6 | `[FEIGN]` | `TxnOrchFeignClient.pay(CardPayOrchRq)` → `POST /txn-orch/card/pay`; pass `X-Member-Id` header; see [txn-orch/card-pay.md](../txn-orch/card-pay.md) |
| 7 | `[RETURN]` | Return `CardPayBffRs { challengeRef, pointsToUse, estimatedTwdAmount }` |

> If Step 6 throws, re-publish operation-log with `FAIL` before propagating the error to the client.

## 7. Error Codes

| HTTP Status | Error Code | Description | Trigger Condition |
|-------------|------------|-------------|-------------------|
| 400 | — | INVALID_REQUEST | Request validation failure |
| 409 | — | DUPLICATE_REQUEST | Idempotency-Key already submitted within 60s |
| 409 | CA00002 | CARD_NOT_ACTIVE | Card is not ACTIVE (propagated: card-service → MockNCCC → ORCH → BFF) |
| 409 | CA00003 | CARD_EXPIRED | Card has expired |
| 409 | CA00004 | CARD_FROZEN | Card is frozen |
| 422 | CA00006 | SINGLE_TXN_LIMIT_EXCEEDED | Transaction exceeds single-transaction limit |
| 422 | CA00007 | DAILY_LIMIT_EXCEEDED | Daily cumulative limit exceeded |
| 503 | CA00013 | COMBINE_KEY_UNAVAILABLE | card-service combineKey not loaded |
| 500 | INTERNAL_ERROR | Unexpected system error | Unhandled exception |
